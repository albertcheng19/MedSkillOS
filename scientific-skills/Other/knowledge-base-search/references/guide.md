# Knowledge Base Retrieval Reference Checklist

## Input Checklist

- Knowledge base path or index entry
- Problem description and keywords
- Filter criteria (time, file type, tags)

## Output Checklist

- Hit list (file name/path/page number or paragraph)
- Hit snippets and brief summaries
- Relevance ranking explanation

## Output Format

- Output the hit list with reference locations.
- Include snippet summaries where necessary.

## Quality Check

- Hit snippets can be traced back to the original file.
- Filter criteria and ranking basis are clear.

## Common Issues

- Providing only conclusions without reference locations.
- Ignoring the update status of the knowledge base.
