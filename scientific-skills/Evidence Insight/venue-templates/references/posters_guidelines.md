# Research Poster Guide

A comprehensive guide to creating effective academic research posters, including dimensions, layout, typography, and design best practices.

**Last Updated**: 2024

---

## Standard Poster Sizes

### International Sizes (ISO 216)

| Size | Dimensions (mm) | Dimensions (in) | Common Uses |
|------|----------------|---------------------|------------|
| **A0** | 841 × 1189 | 33.1 × 46.8 | Most common international standard |
| **A1** | 594 × 841 | 23.4 × 33.1 | Small conferences, easy to carry |
| **A2** | 420 × 594 | 16.5 × 23.4 | Mini posters, small venues |

### US Sizes

| Size | Dimensions (in) | Dimensions (mm) | Common Uses |
|------|-------------------|-----------------|------------|
| **36" × 48"** | 36 × 48 | 914 × 1219 | Common US conference size (Portrait) |
| **42" × 56"** | 42 × 56 | 1067 × 1422 | Large US poster |
| **48" × 36"** | 48 × 36 | 1219 × 914 | Landscape layout |
| **48" × 96"** | 48 × 96 | 1219 × 2438 | Extra-wide format |

### Other Common Sizes

| Size | Dimensions | Notes |
|------|-----------|-------|
| **90 cm × 120 cm** | 900 × 1200 mm / 35.4 × 47.2 in | Common in Europe |
| **40" × 30"** | 1016 × 762 mm | Landscape layout |
| **3 ft × 4 ft** | 914 × 1219 mm / 36 × 48 in | Same as 36×48 |

### Orientation

- **Portrait**: Most common (Height > Width)
  - Best for long visual flow (top-to-bottom)
  - Examples: A0, 36"×48"
- **Landscape**: Less common, but sometimes preferred
  - Best for wide content, timelines
  - Examples: 48"×36", 40"×30"

**Always confirm**: Check specific conference specifications before designing!

## Typography and Font Sizes

### Recommended Font Sizes by Distance

Posters are typically viewed from a distance of 3-6 feet (1-2 meters):

| Element | Size Range | Recommended Size |
|---------|-----------|-------------|
| **Title** | 60-85 pt | 72-85 pt |
| **Author Names** | 48-60 pt | 54 pt |
| **Affiliations** | 32-40 pt | 36 pt |
| **Section Headers** | 36-48 pt | 42 pt |
| **Body Text** | 24-32 pt | 28 pt |
| **Figure Captions** | 20-24 pt | 22 pt |
| **References** | 18-22 pt | 20 pt |

### Font Families

**Sans-Serif (Recommended for posters)**:
- Arial
- Helvetica
- Calibri
- Futura
- Gill Sans
- **Reason**: Clean, easy to read from a distance

**Serif (Use with caution)**:
- Times New Roman
- Georgia
- Palatino
- **Use**: Can be used for body text if preferred, but sans-serif is recommended for headers.

**Monospace**:
- Courier New
- Consolas
- **Use**: Only for code snippets

### Typography Best Practices

1. **Limit Fonts**: Use a maximum of 1-2 font families.
2. **Hierarchy**: Establish a clear size hierarchy.
3. **Weight**: Use bold for emphasis rather than italics.
4. **Alignment**: Left-align body text, center titles.
5. **Spacing**: Use generous line spacing (1.2-1.5).
6. **Consistency**: Use the same font for similar elements.

## Layout and Design Principles

### Grid-Based Layout

**Column Structure**:

| Layout | Columns | Best For |
|--------|---------|----------|
| **Single Column** | 1 | Simple, linear flow; timeline posters |
| **Two Columns** | 2 | Most common; balanced layout |
| **Three Columns** | 3 | Content-dense; multi-part research |
| **Four Columns** | 4 | Extremely dense; avoid if possible |

**Recommendation**: Most research posters use **2 or 3 columns**.

### Visual Flow

**Reading Order** (Western convention):
1. Top-to-bottom
2. Left-to-right
3. Z-pattern or F-pattern path

**Section Ordering** (Typical):
```
+----------------------------------+
|              TITLE               |
|       Authors, Affiliations      |
+----------------------------------+
| Introduction  | Results          |
|               |                  |
| Methods       | Discussion       |
|               |                  |
| [Optional]    | Conclusions      |
+----------------------------------+
|      References / QR Code        |
+----------------------------------+
```

### Spacing and Margins

- **Outer Margins**: Leave 1-2 inches (2.5-5 cm) on all sides.
- **Column Spacing**: 0.5-1 inch (1.3-2.5 cm).
- **Section Spacing**: 0.5-1 inch.
- **White space**: Posters should have 30-40% white space.

**Avoid**: Overly dense text with minimal white space.

## Color Schemes

### Colorblind-Friendly Palettes

Use colorblind-friendly color combinations:

**Okabe-Ito Palette** (Recommended):
- Orange: `#E69F00`
- Sky Blue: `#56B4E9`
- Bluish Green: `#009E73`
- Yellow: `#F0E442`
- Blue: `#0072B2`
- Vermillion: `#D55E00`
- Reddish Purple: `#CC79A7`
- Black: `#000000`
- Gray: `#999999`

**Viridis Palette** (Sequential):
- Suitable for heatmaps and gradients.
- Colorblind-friendly and perceptually uniform.

### Color Usage Guidelines

**Background**:
- **White or Light Gray**: Most common, looks professional.
- **Light Tints**: Pale blue, beige (use with caution).
- **Avoid Dark Backgrounds**: Hard to read, high printing costs.

**Text**:
- **Dark on Light**: Use black or dark gray text on white/light backgrounds.
- **Contrast**: At least 4.5:1 (WCAG AA standard).

**Accent Colors**:
- Use a maximum of 2-3 accent colors.
- Use for section headers, key findings.
- Maintain consistency throughout the poster.

**Figures**:
- Use colorblind-friendly palettes.
- Maintain sufficient contrast.
- Perform grayscale testing.

### Color Contrast Tools
- WebAIM Contrast Checker: https://webaim.org/resources/contrastchecker/
- Coblis (Color Blindness Simulator): https://www.color-blindness.com/coblis-color-blindness-simulator/

## Content Structure

### Essential Sections

#### 1. Title Bar
- **Title**: Clear, specific, and engaging.
- **Authors**: Names (underline the presenter's name).
- **Affiliations**: Institution name, department.
- **Logo**: Institutional logos (usually in corners).
- **Contact**: Email, QR code for paper/website.

#### 2. Introduction/Background
- **Purpose**: Context and motivation.
- **Length**: 100-200 words.
- **Includes**:
  - Problem statement
  - Research gap
  - Objectives/Hypotheses

#### 3. Methods
- **Purpose**: How the study was conducted.
- **Format**:
  - Bullet points preferred.
  - Use flowcharts for complex processes.
  - Key parameters.
- **Includes**: Sample size, steps, analysis methods.

#### 4. Results
- **Purpose**: What was found.
- **Format**: Visually driven (images, tables, charts).
- **Includes**:
  - Key findings (2-4 main results)
  - Statistical significance
  - Visual evidence

#### 5. Discussion/Conclusions
- **Purpose**: Significance of the research.
- **Length**: 100-200 words.
- **Includes**:
  - Interpretation
  - Implications
  - Limitations
  - Future work

#### 6. References
- **Format**: Small font, abbreviated citations.
- **Includes**: Key citations only (5-10).
- **Style**: Just maintain consistency.

### Optional Sections
- **Abstract**: Sometimes included, but often omitted.
- **Acknowledgments**: Funding sources, collaborators.
- **Future Work**: Next steps.

## Visual Elements

### Figures and Plots

**Principles**:
1. **Simplify**: Remove clutter, emphasize the main point.
2. **Enlarge**: Larger than the size used in papers.
3. **Clear Labels**: Large axis labels, legends.
4. **Independence**: Each figure should tell a complete story.
5. **High Resolution**: At least 300 dpi.

**Figure Types**:
- **Photos**: High quality, properly cropped.
- **Statistical Plots**: Bar charts, line graphs, scatter plots.
- **Heatmaps**: Use colorblind-friendly maps.
- **Schematics**: Block diagrams, flowcharts.
- **Tables**: Simple tables (convert complex tables to charts).

### Tables

**When to Use**:
- Precise values are needed.
- Comparisons across different conditions.
- Summary statistics.

**Best Practices**:
- **Keep it Simple**: Max 3-5 columns, 5-10 rows.
- **Large Font**: Same size as body text.
- **Clear Headers**: Bold column/row headers.
- **Row Banding**: Use light shading to improve readability.
- **Minimal Lines**: Keep only horizontal lines (avoid vertical lines).

### Icons and Graphics

**Uses**:
- Use icons to add visual interest (methods, concepts).
- Simple graphics to break up large blocks of text.
- Arrows to guide the reading flow.

**Resources**:
- Noun Project: https://thenounproject.com/
- BioRender: https://biorender.com/ (Scientific illustrations)
- Font Awesome: https://fontawesome.com/ (Icons)

**Note**: Do not overuse; maintain a professional feel.

## LaTeX Poster Packages

### beamerposter

**Description**: Poster extension for Beamer.  
**Best For**: Academic conferences, classic layouts.  
**Pros**:
- Easy for Beamer users.
- Clean, professional interface.
- Many ready-made themes.

**Cons**:
- Less flexible than tikzposter.
- Code can be verbose.

**Template**: `assets/posters/beamerposter_academic.tex`

**Example Usage**:
```latex
\documentclass[final]{beamer}
\usepackage[size=a0,scale=1.24]{beamerposter}
\usetheme{Berlin}
```

### tikzposter

**Description**: Modern poster package using TikZ.  
**Best For**: Colorful, modern designs.  
**Pros**:
- Highly customizable.
- Modern, aesthetic themes.
- Block-based layout.

**Cons**:
- Steeper learning curve.
- Slower compilation.

**Template**: `assets/posters/tikzposter_research.tex`

**Example Usage**:
```latex
\documentclass[25pt, a0paper, portrait]{tikzposter}
\usetheme{Autumn}
\usecolorstyle{Denmark}
```

### baposter

**Description**: Box and poster system.  
**Best For**: Structured, multi-column layouts.  
**Pros**:
- Excellent column control.
- Header boxes, structured layout.
- Good for dense content.

**Cons**:
- Complex syntax.
- Less commonly used.

**Template**: `assets/posters/baposter_conference.tex`

**Example Usage**:

## Printing and File Preparation

### File Formats

**For Printing**:
- **PDF**: The universal standard.
- **High Resolution**: Minimum 300 dpi, 600 dpi recommended for photos.
- **Color Space**: Most printers use RGB (confirm with the print shop).
- **Embed Fonts**: Ensure all fonts are embedded.
- **Flatten**: Avoid issues caused by transparency.

### Printing Quality Check

Before printing:
1. **Proofread**: Check for typos.
2. **Color**: Check in print preview.
3. **Resolution**: Zoom to 100% to check figure quality.
4. **Margins**: Confirm content is not cut off.
5. **Test Print**: Print a small version (A4) first to check the layout.

### Printing Service Providers

**Options**:
1. **University Print Shops**: Usually the cheapest and most convenient.
2. **FedEx Office**: Wide coverage.
3. **Online Services**:
   - Vistaprint
   - Printful
   - Academic Poster Printing (Specialized academic poster printing)

**Cost**: A0 glossy posters are typically around $50-$150.

### Paper Types

| Paper Type | Description | Best For |
|-----------|-------------|----------|
| **Matte** | No reflective coating | Well-lit venues, reduces glare |
| **Glossy** | Shiny, vibrant colors | Photos, colorful charts |
| **Satin/Semi-gloss** | Between matte and glossy | Balanced choice (Recommended) |
| **Fabric** | Wrinkle-resistant, foldable | Travel, multiple uses |

**Recommendation**: Most academic posters choose **satin or matte**.

## QR Codes

### What to Include

Generate QR codes linking to:
- **Paper PDF**: Published version or preprint.
- **Supplementary Materials**: Data, code, videos.
- **Personal Website**: Lab or researcher homepage.
- **Video Abstract**: 1-2 minute video introduction.
- **Online Version**: Interactive poster.

### Placement

- **Common Locations**:
  - Bottom right corner.
  - Next to references.
  - Near contact information.
- **Size**: 3-4 inches (7-10 cm) square.
- **Label**: Mark as "Scan for paper" or "More information."

### QR Code Generators
- QR Code Generator: https://www.qr-code-generator.com/
- QRStuff: https://www.qrstuff.com/
- LaTeX Package: `qrcode` for generating within LaTeX.

## Design Best Practices

### Do's

✓ **Use large fonts** (Body text ≥28pt).  
✓ **Keep text minimal** (30-40% of the poster).  
✓ **Use visual elements** (60-70% figures, images).  
✓ **Tell a story** (Clear narrative flow).  
✓ **Colorblind-friendly colors**.  
✓ **Test readability** (View from 6 feet away).  
✓ **Include contact info** (Email, QR code).  
✓ **Proofread multiple times**.

### Don'ts

✗ **Do not use small fonts** (Body text <24pt).  
✗ **Do not overcrowd** (Leave white space).  
✗ **Do not use complex tables** (Simplify or visualize).  
✗ **Do not use full paragraphs** (Use bullet points).  
✗ **Do not use too many fonts** (Max 1-2).  
✗ **Do not use low-res images** (<300 dpi).  
✗ **Do not use red-green contrast** (Colorblindness issues).  
✗ **Do not copy-paste your paper** (Poster ≠ Paper).

## Poster Presentation Tips

### During the Poster Session

1. **Stay by your poster**: Be present and actively engaged.
2. **Prepare an elevator pitch**: Have a 1-2 minute short introduction ready.
3. **Differentiate depth**: Short version (1 min), medium version (3 mins), deep version (10 mins).
4. **Engage the audience**: Ask questions and invite discussion.
5. **Business cards**: Carry them with you.
6. **Notebook**: Record feedback and questions.
7. **Handouts**: Optional, prepare a 1-page summary (with QR code).

### Poster Presentation Logic

**30-Second Version**:
- What is the problem?
- What did you do?
- What did you find?

**2-Minute Version**:
- Background + Motivation.
- Methods (briefly).
- Key results (show main figures).
- Conclusions + Significance.

**5+ Minute Version**:
- Full walkthrough of the process.
- Answer specific questions.
- Discuss limitations and future work.

## Accessibility Considerations

### Visual Accessibility

1. **Color Contrast**: High contrast (minimum 4.5:1).
2. **Colorblind-friendly**: Use Okabe-Ito or similar palettes.
3. **Font Size**: Large enough for distance reading.
4. **Font Choice**: Clear sans-serif fonts.
5. **Alt Text**: Consider providing text descriptions.

### Physical Accessibility

1. **Hanging Height**: Low enough for wheelchair users to read the bottom content.
2. **QR Codes**: Provide alternatives (short links, handouts).

## Pre-Printing Checklist

- [ ] Proofread all text (spelling, grammar).
- [ ] Check author names and affiliations.
- [ ] Confirm all figures are high resolution (300+ dpi).
- [ ] Ensure colorblind-friendly color scheme is used.
- [ ] Test readability from 6 feet away (print a sample).
- [ ] Confirm poster size meets conference requirements.
- [ ] Check if fonts are embedded in the PDF.
- [ ] Include contact info (email, QR code).
- [ ] Add institutional logos.
- [ ] Verify references are accurate.
- [ ] Ensure figures have clear labels and captions.
- [ ] Check if layout is too crowded (sufficient white space).
- [ ] Confirm QR code works (scan test).
- [ ] Confirm file is a high-resolution PDF.
- [ ] Get feedback from colleagues.

## Summary

**Key Takeaways**:

1. **Dimensions**: Verify conference requirements (usually A0 or 36"×48").
2. **Fonts**: Large fonts (Body ≥28pt, Title ≥72pt).
3. **Layout**: 2-3 columns with ample white space.
4. **Visuals**: Visual content should occupy 60-70%.
5. **Color**: Colorblind-friendly, high contrast.
6. **Content**: Tell a story, keep text concise.
7. **Quality**: 300+ dpi, perform a test print.
8. **Accessibility**: Readable from a distance, clear hierarchy.

**Remember**: A poster is **not a paper** — it is a visual summary designed to spark conversation!
